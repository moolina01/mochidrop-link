## COMPARACIÓN — Antes vs. Después

**Encabezado de sección:**
> Etiqueta: Antes vs. Después
> Título: El mismo envío. Dos realidades.

**Sin MochiDrop (columna izquierda):**
1. ✗ Buscas el precio del envío en la web del courier
2. ✗ Vuelves a WhatsApp a explicar las opciones
3. ✗ Esperas que el cliente transfiera (si es que vuelve)
4. ✗ Generas la guía a mano en la web del courier
5. ✗ Copias y pegas el tracking al cliente

**Con MochiDrop (columna derecha):**
1. ✓ Creas el link con los datos del cliente
2. ✓ Lo mandas por WhatsApp
3. ✓ Tu cliente elige, paga con tarjeta y listo
4. ✓ La guía se genera automáticamente
5. ✓ El tracking llega solo al cliente

**Barra inferior (métricas):**
- ~15 min → por envío manual
- ~30 seg → por envío con MochiDrop
- **NUEVO — Botón CTA:** Recuperar mi tiempo →

### Notas de implementación:
- Cambié "Cotizas envío en cada courier manualmente" por "Buscas el precio del envío en la web del courier" — esto refleja la realidad de las PYMEs que usan UN courier, no varios.
- "si es que vuelve" en el punto 3 es una puñalada emocional sutil.
- El CTA en la barra inferior captura al usuario en su momento de mayor convicción.

## CALCULADORA DE TIEMPO

**Encabezado de sección:**
> Etiqueta: Calculadora de tiempo
> Título: El tiempo que pierdes tiene un precio

**Pregunta:** ¿Cuántos envíos haces a la semana?
**Input:** [número] envíos / semana

**Resultados:**
| Métrica | Valor |
|---------|-------|
| Tiempo manual por semana | X min |
| Tiempo con MochiDrop | X min |
| **Tiempo que recuperas al mes** | **Xh** |
| **Ventas extras que podrías cerrar** | **~X ventas** |

### Notas de implementación:
- Cambié el título de "¿Cuánto tiempo estás perdiendo?" (que suena a regaño) a "El tiempo que pierdes tiene un precio" (que suena a oportunidad).
- NUEVA FILA: "Ventas extras que podrías cerrar" — calcular como: (horas ahorradas × 2 ventas/hora). Esto traduce tiempo → dinero/oportunidad.
- Fórmula sugerida: ventasExtras = Math.round((savedHours) * 2)
